# DevDesk 方案与开发计划

## 1. 项目定位

DevDesk 是一个本地单用户桌面应用，用于管理个人开发学习过程：

- 管理开发项目
- 记录学习日志
- 维护任务
- 登记后续小应用入口

第一阶段目标是做出一个清晰、能用、方便继续迭代的个人工作台，不追求复杂项目管理系统。

## 2. 技术路线

当前选择：

- 桌面端：Electron
- 前端：React + TypeScript
- 构建：Vite
- 包管理：pnpm
- 样式：普通 CSS
- 当前存储：localStorage
- 后续存储：SQLite

暂不做：

- 登录
- 云同步
- 多用户
- 插件系统
- 拖拽看板
- 移动端

## 3. MVP 页面

第一版包含 6 个页面：

- Dashboard：首页概览，展示项目数、任务数、日志数、最近项目、当前任务、最近日志
- Projects：项目管理，支持项目新增、编辑、删除
- Logs：学习日志，支持日志新增、编辑、删除
- Tasks：任务管理，支持任务新增、状态切换、删除
- Apps：应用入口，登记网页链接、本地路径或后续内置页面
- Settings：设置入口，先保留主题、备份、数据存储说明

## 4. 核心数据

项目 `Project`：

- 名称
- 状态：想法中、开发中、暂停、完成
- 技术栈
- 当前目标
- 下一步
- 本地路径
- 仓库地址

学习日志 `LearningLog`：

- 日期
- 关联项目
- 今天学了什么
- 遇到的问题
- 解决方案
- 明天继续

任务 `Task`：

- 标题
- 关联项目
- 状态：待做、进行中、完成
- 优先级：低、中、高
- 截止日期
- 备注

应用入口 `AppEntry`：

- 名称
- 类型：网页链接、本地路径、内置页面
- 描述
- 地址或路径

## 5. 架构原则

当前数据流：

```text
React 页面
  -> useWorkspaceData.ts
  -> storage.ts
  -> localStorage
```

后续 SQLite 数据流：

```text
React 页面
  -> 前端 service / hook
  -> preload API
  -> Electron main
  -> SQLite
```

约定：

- React 页面不直接访问 SQLite
- 页面组件不直接读写底层存储
- Electron API 只通过 `preload` 暴露必要能力
- 数据操作集中在 hook/service 层，方便后续迁移 SQLite

## 6. 阶段计划

### 阶段 1：基础工作台

目标：

- Electron 窗口能启动
- React 页面正常显示
- 六个基础页面可切换
- 使用 localStorage 保存数据

验收：

- 能新增项目、日志、任务、应用入口
- 关闭重开后数据仍在

### 阶段 2：完善基础 CRUD

目标：

- 项目、日志、任务、应用入口都支持新增、编辑、删除
- 表单体验更清晰
- 页面组件逐步拆分

验收：

- 主要数据都能完整维护
- 代码结构不再集中在单个超大文件

### 阶段 3：导入导出

目标：

- 导出 JSON 备份
- 导入 JSON 恢复

验收：

- 用户能手动备份 localStorage 数据
- 迁移 SQLite 前有安全兜底

### 阶段 4：SQLite

目标：

- 在 Electron 主进程接入 SQLite
- 建立 Projects、Logs、Tasks、Apps 表
- 前端通过 preload API 读写数据

验收：

- 数据保存到明确的本地数据库文件
- 关闭重开后数据稳定保留

### 阶段 5：桌面能力和打包

目标：

- 打开本地项目路径
- 打开网页链接
- 打包 Windows 桌面应用

验收：

- 能生成可运行的 Windows 应用

## 7. 当前优先级

近期建议顺序：

1. 完成任务编辑功能
2. 完成应用入口编辑功能
3. 拆分 `App.tsx`
4. 增加 JSON 导出 / 导入
5. 接入 SQLite
