# AI Vibe Coding 学习推进指南

## 1. 核心原则

用 AI 写代码时，不把目标设为“让 AI 一次写完”，而是设为：

> 借助 AI 推进项目，同时把每次改动变成一次学习。

每次只做一个小功能，做完就运行、检查、提交、复盘。

## 2. 推荐工作流

每个功能按这个流程推进：

1. 明确一个小目标
2. 先解释方案，不写代码
3. 再实现代码
4. 运行检查
5. 查看 Git diff
6. 讲解关键代码
7. 总结本次知识点
8. 提交 Git

## 3. 推荐提示词

开始一个功能时：

```text
我们做 [功能名]。
要求：
1. 先解释方案，不写代码。
2. 再实现。
3. 实现后告诉我改了哪些文件。
4. 最后总结我应该学习的知识点。
```

如果想自己参与一部分：

```text
这个功能你先写一半，剩下的步骤告诉我，我自己补。
```

如果没看懂：

```text
这段代码我没看懂，用初学者能懂的方式解释。
```

如果想复盘：

```text
根据这次 git diff，帮我讲解本次改动。
```

如果想测试理解：

```text
先不要继续写，出 5 个问题考我一下。
```

## 4. 每次学习要问的问题

每完成一个功能，可以问：

- 这次改动解决了什么问题？
- 改了哪些文件？
- 哪些代码是核心？
- 我应该重点理解哪些 React / TypeScript / Electron 知识？
- 有没有更简单或更规范的写法？
- 这次适合怎么写 commit message？

## 5. DevDesk 推荐学习路线

按下面顺序逐步学习：

1. React 组件和 JSX
2. `useState` 和状态更新
3. `useEffect` 和副作用
4. props 和 TypeScript 类型
5. 表单处理
6. 列表渲染和条件渲染
7. 组件拆分
8. `localStorage` 本地保存
9. Electron 主进程和渲染进程
10. preload 和 IPC
11. SQLite 本地数据库
12. 打包发布 Windows 桌面应用

## 6. Git 学习节点

每完成一个清晰功能就提交一次。

示例：

```powershell
git add .
git commit -m "feat: add project editing"
```

提交后可以复盘：

```text
根据刚才的提交，解释这次我应该学会什么。
```

## 8. 学习时的分工

用户负责：

- 运行项目
- 观察界面结果
- 提出不懂的地方
- 操作 Git 提交
- 尝试补一小部分代码

AI 负责：

- 拆解方案
- 编写或修改代码
- 解释关键代码
- 指出本次知识点
- 帮助复盘 Git diff

目标不是追求快，而是让项目每推进一步，你都能真正理解一点。
